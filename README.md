# Проект: KDA-Library (Keystroke Dynamics Authenticator)

## Цель проекта:

Создание системы статической аутентификации на основе динамики нажатий клавиш. В отличие от классических паролей, верификация происходит не по тому, что вводит пользователь, а по тому, как он это делает — ритм, тайминги, характерные паттерны — это называется клавиатурный почерк. Подделать клавиатурный почерк другого человека практически невозможно, на этом и основан принцип работы аутентификатора.
Пользователь вводит заданную фразу 30-40 раз в режиме регистрации. Система строит поведенческий профиль и затем верифицирует его при каждом входе.

## Задачи проекта

### 1. Клиентская часть (Frontend SDK)

* **Сборщик событий (Event Capturer):** Разработка обработчика, который фиксирует события при вводе текста
* **Режим обучения (Enrollment):** Логика циклического ввода (30-40 раз) для формирования первичного датасета пользователя.
* **Селектор устройств:** Механизм переключения пресетов (Desktop/Mobile), так как физика ввода на сенсоре и механике принципиально разная.
* **Адаптивный экспорт:** Формирование нормализованного вектора признаков в формате JSON для отправки на верификацию.

### 2. Аналитический модуль (Python AI)

* **Feature Engineering:** Преобразование сырых таймингов в вектор признаков:
* **Hold Time ():** Длительность удержания каждой клавиши.
* **Flight Time ():** Интервалы между отпусканием  и нажатием .
* **Digraphs:** Среднее время перехода для связок символов (например, `@` и `g` в почте).


* **Модель классификации:** Реализация **One-Class SVM** или **Random Forest**. Задача модели — определить, входит ли текущий вектор в доверительный интервал «эталонного» почерка владельца.
* **Оценка точности:** Расчет метрик **FAR** (ложное допущение) и **FRR** (ложный отказ) на тестовой выборке.

### 3. Интеграция в другие сервисы

```typescript
import { KDA } from 'kda-sdk'
 
const kda = new KDA({ endpoint: 'https://your-kda-server.com' })
kda.attach(document.getElementById('login-input'))
 
// При регистрации
await kda.enroll(userId)
 
// При входе
const result = await kda.verify(userId)
// -> { verified: true, confidence: 0.87 }
```

## Спецификация MVP (Minimum Viable Product)

### Основной функционал:

1. Регистрация почерка
Пользователь вводит заданную фразу 30-40 раз. SDK накапливает события, формирует датасет попыток и передаёт его в аналитический модуль для построения профиля.
 
2. Верификация
Пользователь вводит фразу несколько раз. Система сравнивает тайминговый вектор с эталоном и возвращает вердикт с уровнем уверенности.
 
3. Селектор устройств
Переключатель Desktop / Mobile. Физика ввода на сенсоре и механической клавиатуре принципиально различается — модель учитывает это при верификации.
 
4. Демо (GitHub Pages)
Интерактивная страница с визуализацией процесса: обучение -> верификация -> вердикт. Хранение профиля в LocalStorage.

### Технологический стек:

* **Frontend:** TypeScript (сбор данных, UI).
* **Data Science:** Python (Pandas, Scikit-learn) — для обучения модели и генерации весов.
* **Storage:** LocalStorage (хранение профиля почерка в браузере для демо-целей).
